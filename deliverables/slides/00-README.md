# Client Meeting Deck — Index

**Engagement:** WhatsApp Operations Agent for a Hyderabad cleaning services SMB
**Prepared by:** Keerthan Venkata, TinKern Labs
**Date:** 22 April 2026

This folder is the deck content, one slide per Markdown file. Each file is structured to be dropped into PowerPoint/Google Slides/Keynote as-is:

- **Slide Title** — what appears at the top
- **Objective** — what this slide must achieve in the conversation (for you)
- **Key Points** — bullet content for the slide itself
- **Visual / Diagram** — what to put on the slide visually
- **Speaker Notes** — what you actually say
- **Questions to Ask** — discovery questions for this topic
- **What to Capture** — what you must note during the discussion

## Style rules

- No hype. No jargon. If a term appears, define it in one line on the slide or in speaker notes.
- Owner-pain first. Every slide should tie back to "less chaos, fewer missed leads, owner off the hot seat."
- Keep bullet lists to 4–6 items max on a slide. Expand via speech, not text.
- Prices are in INR. USD conversions only in speaker notes when useful.
- All numbers sourced from research dated April 2026; see `playbook/07-recommended-stack.md` for citations.
